================================================================================
  CONFIGURATION KEYS
================================================================================

SMTP_HOST           → SMTP server hostname (e.g., smtp.gmail.com)
SMTP_PORT           → SMTP port (587 for TLS, 465 for SSL)
SMTP_USER           → SMTP username or email
SMTP_PASSWORD       → SMTP password (automatically masked in frontend)
SMTP_FROM_EMAIL     → Sender email address
SMTP_SECURE         → 1 for TLS/SSL, 0 for no encryption

================================================================================
  CONFIGURATION PRIORITY
================================================================================

When loading a configuration value:

1. Check in-memory cache (fastest)
2. Query database (if not cached)
3. Fall back to environment variables
4. Use default value (if provided)

Example flow for SMTP_HOST:
  AdminConfig::getOption('SMTP_HOST', 'default.com')
  → Check cache (hit? return)
  → Query database (found? cache and return)
  → Check $_ENV['SMTP_HOST'] (exists? cache and return)
  → Return 'default.com'

================================================================================
  ADDING NEW GLOBAL OPTIONS
================================================================================

No database migration needed! Just use:

  AdminConfig::updateOption('MY_NEW_OPTION', 'value');

Later, get it back:

  $value = AdminConfig::getOption('MY_NEW_OPTION', 'default');

That's it! The system handles everything.
